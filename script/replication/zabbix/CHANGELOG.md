# Changelog

## [v1] - 2026-04-16
### Changed

1. Обновлены zabbix-скрипты мониторинга отказоустойчивости.
2. Изменён путь для zabbix-скриптов мониторинга: `script/replication/zabbix/v1/*`
3. Проверьте права по пути до zabbix-скриптов:
    - `namei -l <путь к инсталлятору>/script/replication/zabbix/v1`
4. Если увидите отсутствующие права (например, `drwx------` или отсутствие доступа для других пользователей) - исправьте права доступа для возможности выполнения zabbix-агентом:
    - Добавьте права на выполнение для тех директорий, где они отсутствуют: `chmod o+x <путь к директории>`
    - Для скриптов в директории: `chmod +x script/replication/zabbix/v1/*.py`
5. Измените путь zabbix-скриптов в конфигурационном файле zabbix-агента и выполните рестарт:
    - Откройте файл для редактирования: `vim /etc/zabbix/zabbix_agentd.conf`
    - Измените путь скриптов на: `script/replication/zabbix/v1/*`
    - Перезапустите агент: `systemctl restart zabbix-agent`
